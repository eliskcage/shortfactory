Parameter Playbook

Every constant. What it does. What breaks first.

ParameterDefaultEffectBreak point
RESONANCE0.75 Cosine similarity threshold for a signal to activate a sphere. Higher = stricter matching. >0.92: signals miss existing spheres constantly, spawns explosion. <0.5: everything resonates with everything, no concept isolation.
MERGE_SIMILARITY0.88 How similar two grooved spheres must be before they merge into a child concept. >0.95: merges almost never fire, concepts accumulate and decay without synthesis. <0.75: cascade explosion — everything merges with everything.
GROOVE_TO_MERGE5 How many signals a sphere must receive before it becomes merge-eligible. <3: premature merges from noise signals. >15: concepts take too long to stabilise, short-lived inputs never synthesise.
DRIFT_RATE0.06 How much the sphere's embedding vector moves toward each incoming signal. >0.2: sphere becomes unstable, chases recent signals instead of averaging. <0.01: sphere never moves, loses ability to learn new nuance.
BASE_TIMER60 Ticks before an unreinforced sphere dissolves. Extended by groove depth (groove × 3). <20: system forgets too fast, nothing survives long enough to merge. >200: stale concepts persist, pollute concept space.
MAX_MERGES_TICK10 Governor — maximum merges allowed per tick cycle. Remove this: cascade explosion. One merge triggers two ready parents, they merge, triggers four, system locks up generating thousands of child concepts per second.
DECAY_LONELY1.6× Isolated spheres (no neighbours within cosine 0.5) decay faster than this multiplier. >3.0: isolated concepts dissolve before they can be reinforced. <1.0: isolation has no penalty, concept space fills with orphaned nodes.

Tuning for your use case

Fast-moving data (news, social): Lower BASE_TIMER (30-40), lower GROOVE_TO_MERGE (3), higher DECAY_LONELY (2.0). The system forgets quickly and only retains what's being actively reinforced.

Stable knowledge base (medical, legal): Higher BASE_TIMER (120+), higher GROOVE_TO_MERGE (10), lower DRIFT_RATE (0.02). Concepts are conservative — they require extensive reinforcement before merging and they don't forget quickly.

Language learning / translation: RESONANCE at 0.70, MERGE_SIMILARITY at 0.85. Lower thresholds allow synonyms and near-equivalents to converge into shared concepts across languages.

Industrial Wiring Guide

From local demo to production system.

Step 1 — Install Qdrant (self-hosted)

# Docker — persistent vector store
docker pull qdrant/qdrant
docker run -d -p 6333:6333 -p 6334:6334 \
  -v $(pwd)/qdrant_storage:/qdrant/storage:z \
  qdrant/qdrant

# Then in spherenet_industrial.py, change:
qdrant = QdrantClient(":memory:")
# to:
qdrant = QdrantClient(host="localhost", port=6333)

Step 2 — Decouple the tick engine

Running do_tick() on every signal call bottlenecks at ~1k signals/sec. For production, decouple:

# Install Redis + Celery
pip install celery redis

# celery_app.py
from celery import Celery
from celery.schedules import crontab

app = Celery('spherenet', broker='redis://localhost:6379/0')

app.conf.beat_schedule = {
  'tick-every-500ms': {
    'task': 'tasks.do_tick',
    'schedule': 0.5,  # every 500ms
  },
}

# Run: celery -A celery_app worker --beat

Step 3 — Scale the embedding layer

The default encoder (all-MiniLM-L6-v2) runs on CPU at ~100 embeddings/sec. For higher throughput:

# GPU acceleration (if you have CUDA)
encoder = SentenceTransformer('all-MiniLM-L6-v2', device='cuda')
# → ~5,000 embeddings/sec on RTX 4090

# Or batch signals before embedding
texts = ["signal 1", "signal 2", "..."]
vecs = encoder.encode(texts, batch_size=64, normalize_embeddings=True)

Step 4 — Merge governor tuning

At scale, the merge governor is critical. The default MAX_MERGES_TICK=10 is conservative. Tune it by monitoring:

# Watch for cascade signals:
if merge_count > MAX_MERGES_TICK * 0.8:
    print("WARNING: merge governor near limit — consider raising GROOVE_TO_MERGE")

# Scale governor with sphere count:
MAX_MERGES_TICK = max(10, len(spheres) // 500)

Step 5 — Lineage querying at scale

# For deep lineage queries, add PostgreSQL
# On every merge, write an edge:
INSERT INTO lineage (child_id, parent_a_id, parent_b_id, tick, similarity)
VALUES (%s, %s, %s, %s, %s);

# Query: full ancestry of a concept
WITH RECURSIVE ancestry AS (
  SELECT child_id, parent_a_id, parent_b_id, 0 AS depth
  FROM lineage WHERE child_id = %s
  UNION ALL
  SELECT l.child_id, l.parent_a_id, l.parent_b_id, a.depth + 1
  FROM lineage l JOIN ancestry a ON l.child_id IN (a.parent_a_id, a.parent_b_id)
  WHERE a.depth < 20
)
SELECT * FROM ancestry;
Safe AGI Audit Protocol

How to verify a SphereNet is telling the truth.

The architecture guarantees honesty by physics — but you still need a protocol to audit it. Here's the checklist:

  • Check for stale high-groove concepts — any sphere with groove > 20 and timer > 200 has been heavily reinforced. Query its lineage. Verify the signal history matches expected input sources.
  • Check for isolated high-generation concepts — a gen-5+ concept with no neighbours is a red flag. It formed from merges but nothing is reinforcing it now. May represent a belief the system is "holding on to" without evidence.
  • Inject known-false signals — send signals you know are incorrect ("the sun is cold"). The system should either fail to resonate (good) or create a new sphere that decays quickly (acceptable). If it resonates with a high-groove concept and boosts it — the RESONANCE threshold is too low.
  • Check timer distribution — in a healthy system, timer values should be spread across the range. If most spheres are near BASE_TIMER, the system isn't learning. If most are near 0, it's forgetting too fast.
  • Verify merge lineage integrity — every merged concept should have exactly two parents. Any concept with no parents that isn't a seed is a ghost — created by a bug.
  • Run a classify round-trip — signal a concept, immediately classify similar text. Confidence should be >0.7. If it's below 0.5, RESONANCE is misconfigured or the embedding model doesn't cover this domain.

Worked Examples

Seed your own concept spaces.

Emotion space

# Seed the eight primary emotions at cardinal positions
net.seed("joy happiness delight pleasure elation", "JOY")
net.seed("sadness grief sorrow loss despair", "SADNESS")
net.seed("anger rage fury outrage irritation", "ANGER")
net.seed("fear dread terror anxiety panic", "FEAR")
net.seed("disgust revulsion contempt loathing", "DISGUST")
net.seed("surprise shock astonishment wonder", "SURPRISE")
net.seed("trust confidence faith reliability", "TRUST")
net.seed("anticipation expectation eagerness hope", "ANTICIPATION")

# Now signal ambiguous emotional states
net.signal("nervous excitement before a performance")
# → emerges at FEAR+ANTICIPATION boundary

net.signal("bittersweet nostalgia for something lost")
# → emerges at JOY+SADNESS boundary
